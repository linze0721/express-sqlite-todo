# PRD: Simple Todo List (Express + SQLite + Vanilla HTML)

## Summary

Public todo application with a Node.js/Express 4 API backed by SQLite (`better-sqlite3`), and a minimal single-page UI in plain HTML + vanilla JavaScript. No authentication. Full page reload after each user action (not an SPA).

## Goals

- Let anyone create, list, toggle completion, and delete todos without signing in.
- Keep the stack small and easy to run locally.

## Out of scope (v1)

- User accounts / login
- Automated tests
- Separate frontend build tooling or frameworks
- Deployment automation (local dev only unless specified later)
- Pagination, search, due dates, priorities, tags

## Tech stack

| Layer | Choice |
|-------|--------|
| Runtime | Node.js |
| HTTP | Express 4 |
| Database | SQLite via `better-sqlite3` |
| Schema | `CREATE TABLE IF NOT EXISTS` on server startup (no separate migration tool) |
| Frontend | Static HTML + vanilla JS served by the same Express app |

## Data model

**Table: `todos`**

| Column | Type | Notes |
|--------|------|--------|
| `id` | INTEGER PRIMARY KEY AUTOINCREMENT | |
| `title` | TEXT NOT NULL | User-provided |
| `done` | INTEGER NOT NULL DEFAULT 0 | 0 = false, 1 = true |
| `created_at` | TEXT NOT NULL | ISO 8601 UTC, set by server on create |

## API

Base path: `/api/todos`

| Method | Path | Behavior |
|--------|------|----------|
| `GET` | `/api/todos` | List all todos, newest first (`ORDER BY created_at DESC` or `id DESC`) |
| `POST` | `/api/todos` | Body: `{ "title": string }`. Validates non-empty trimmed title. Sets `done = 0`, `created_at` now. Returns created row. |
| `PATCH` | `/api/todos/:id` | Body: `{ "done": boolean }`. Updates done flag. 404 if missing id. |
| `DELETE` | `/api/todos/:id` | Removes row. 404 if missing id. |

**Errors:** JSON `{ "error": "message" }` with appropriate HTTP status (400 validation, 404 not found, 500 server).

**CORS:** Not required if UI is same-origin; same Express app serves static files and API.

## Frontend UX

- **Layout:** Max width 600px, centered; system font stack; clean, minimal spacing.
- **Top:** HTML form with text input + submit to add a todo (`POST` via form or fetch then reload).
- **Below:** List of todos; each row shows title, checkbox for `done`, and **X** to delete.
- **Interactions:** Checkbox toggles `done` via `PATCH`; X deletes via `DELETE`. After each successful action, **reload the full page** (no client-side routing or SPA state).
- **Server:** Express serves `public/` (or equivalent) for `index.html` and assets; API on same port.

## Server startup

1. Open SQLite database file (e.g. `data/todos.db` or project-relative path).
2. Run `CREATE TABLE IF NOT EXISTS todos (...)` with columns above.
3. Listen on configurable port (default e.g. 3000).

## Acceptance criteria

- [ ] `npm start` (or documented command) starts API + static UI.
